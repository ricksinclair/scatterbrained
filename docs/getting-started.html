scatterbrained getting started

getting started

From zero to your own observatory.

No prior graph-database knowledge needed. Here's the plain-language version of what Scatterbrained is, how to install it in about five minutes, and what to do once it's open.

you'll need: a Mac, Windows, or Linux machine · ~5 minutes · no account, nothing in the cloud

first — the words

A 60-second glossary.

A few terms you'll see. None of them require you to be a database person.

Knowledge graph
Your notes, decisions, files, and the people and projects they touch — stored as things and the links between them, so you can follow a thread instead of searching for a file.
Neo4j
The local database that holds the graph. Scatterbrained starts one for you in Docker; you never have to touch it. It runs on your machine — your data never leaves.
The Studio
The app you actually use: a browser window that draws your graph as a living constellation you can explore, read, and edit.
The agent
Claude, working in your projects. Scatterbrained gives it a CLAUDE.md and a couple of hooks so it consults the graph first and keeps it tidy — see the system.
Protected facts & notes
Pin a number or citation so a rewrite can't quietly change it (with an optional detail label); jot a note on anything and the next sync has the agent act on it. Two different things — explained here.

install — macOS

Two things to install once. Then one command.

Homebrew installs Node and Docker; npm start does the rest — it starts the database, loads a demo graph, and opens the Studio.

  1. Install Homebrew

    Skip if you already have brew. Otherwise paste into Terminal:

    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

  2. Install Node and Docker

    brew install node
brew install --cask docker

    then open Docker once from Applications so it's running. No-GUI alternative: brew install colima docker then colima start.

  3. Get Scatterbrained and run it

    git clone https://github.com/ricksinclair/scatterbrained.git
cd scatterbrained
npm install
npm start          # → http://localhost:4317

    The Studio opens with a demo graph loaded. If anything's missing, npm start tells you exactly what to install.

    or already have Node + Docker? Skip the clone — one command does everything (download, database, demo, launch):

    npx scatterbrained@alpha studio          # → http://localhost:4317

    tip or install it once — npm i -g scatterbrained@alpha — then run scatterbrained studio (and scatterbrained help for the toolkit) from anywhere.

Windows

winget install OpenJS.NodeJS Docker.DockerDesktop, launch Docker Desktop, then npm install && npm start.

Linux

Install Node 18+ and Docker Engine, then npm install && npm start. Already run a Neo4j 5? Set NEO4J_PASSWORD and it'll use that.

your first five minutes

Now that it's open.

  1. Explore the demo constellation

    It opens on a small fictional engineering team's graph. Drag to pan, scroll to zoom, and click any node — the inspector on the right builds itself from what that node is.

  2. Take the guided tour

    Open http://localhost:4317/#tour for a narrated auto-play through the headline features. Stop it anytime.

  3. Point it at your own graph

    When you're ready, run NEO4J_PASSWORD=… npm start against your own Neo4j (or keep using the local one). A fresh graph starts empty — that's expected; you fill it as you work.

  4. Stand up a project

    One command sets up a project in the graph and drops a wired CLAUDE.md in your repo (npm run new-project -- --name "My Project") — or just ask Claude. See the honest caveats on what it writes first.

two ways in

Type a command, or just ask.

Everything Scatterbrained does is reachable both ways. Same result — pick your lane.

Ask Claude

If you use Claude with the Neo4j (and optional Notion) connection, just say what you want in plain language. No terminal, no token.

"Where were we on the shipment project?"

"Set up a new Scatterbrained project called Acme."

The CLI

Prefer the keyboard? The scatterbrained (sb) command wraps the toolkit.

sb resume · sb search "how this makes money"

sb new-project --name "Acme" · sb lint:graph

honest, up front

Scatterbrained is alpha. AI summaries / Q&A are optional and run on a local Ollama — everything else works without it, and nothing leaves your machine. Before you wire it into a real project, skim what it changes on your machine.
