getting started

Welcome. Let's get you set up.

You don't need to be a developer. If you can use Notion and chat with Claude, you can run Engram. This page walks you all the way from nothing to a working setup — in plain English, one step at a time.

By the end you'll have two things working together: an AI memory that remembers your projects, notes, and decisions across conversations — and a ready-to-use Notion workspace to actually run your work (a task board, a changelog, and more). Set up together, as one system.

first — what is this, really?

Four words, in plain English.

You'll see these terms a lot. Here's what each one actually means — no jargon.

Knowledge graph
A way of storing facts as connected dots — "this project needs this skill," "this decision came from this note." Because the dots are linked, an assistant can follow the connections and answer questions a plain list never could. It's the part that remembers.
Capture lane
Wherever you naturally jot things down — a Notion page, a document on your computer. Engram reads those and feeds the facts into the graph for you. You write where it's comfy; it files everything where it's useful.
Operations workspace
The Notion side where you actually run a project: a task board (Kanban), a changelog, test tracking, a docs index. Engram sets this up for you, ready to go.
The one root
A single Notion page that becomes the home for all your projects. You share it with the assistant once, and everything you add later lives under it automatically. Set up once, never fuss again.

track a — notion + claude · no terminal

Just ask Claude.

In this path, Claude does the setup. You connect it to Notion once, give it one home page to work in, and then ask — in normal English — for what you want.

  1. Connect Claude to Notion

    In Claude, turn on the Notion connection (Settings → Connectors → Notion, then approve access). This is what lets Claude read and build pages for you. One-time, a couple of clicks.

  2. Make one home page — your “root”

    In Notion, create a single page and name it something neutral like “Projects” or “Workspace.” Open its ••• menu → Connections → add your Claude/Notion connection. You share this one page, once — every project you add later nests under it and inherits access automatically.

    Why one root? So you never have to share pages again, and so all your projects live in one place the assistant can reason across.

  3. Ask Claude to set up a project

    Now just say it. For example:

    “Set up a new Engram project called Acme under my Projects page.”

    Point Claude at Engram's recipe so it follows the exact steps — skills/new-project.md. It will create the workspace, register the project in the graph's memory, and add a settings file to your repo — all over the connection you just approved.

  4. See what you got

    Open Notion and look under your root page. You'll find a new Acme — Workspace with a task board, a changelog, test tracking, a docs index, and a Capture page (drop notes there and they flow into the memory). The assistant now remembers Acme across every future chat.

  5. Live with it — just ask

    From here, everything is a request in plain language. A few you'll use often:

    • “Where were we?” — a catch-up brief across all your projects.
    • “Run a graph-sync.” — re-read what changed, file new facts, back up.
    • “Search Engram for how Acme makes money.” — find by meaning, not just keywords.

    See them all on the Skills section.

one honest note The memory itself (a small database called Neo4j) has to run somewhere — usually a single one-time install on a computer, started with one command (docker compose up -d). It's the one slightly technical bit. You can do it once by following the developer steps below, or have a tech-savvy friend (or Claude, walking you through it) set it up once — then never think about it again.

track b — developer · cli & token

Set it up from the terminal.

The scriptable path. Same result as Track A — a Notion workspace plus a project in the graph — driven by commands instead of conversation. Ideal for automation, CI, or if you just prefer the keyboard.

  1. Clone it and start the graph

    Get the code and spin up Neo4j locally (this is the “memory” database). The demo credentials are fine as-is for local use.

    git clone https://github.com/ricksinclair/engram.git && cd engram
npm install
cp .env.example .env
docker compose up -d                # Neo4j now at http://localhost:7474

  2. Create a Notion integration token

    At notion.so/my-integrations, create an internal integration and copy its secret. Then, in Notion, share your one root page with that integration (page ••• → Connections). Put both values in .env:

    # .env
NOTION_TOKEN=secret_xxxxxxxxxxxxxxxxxxxx
NOTION_PARENT_PAGE_ID=your_root_page_id

    The root page id is the long string in the page's URL. Everything you create nests under it — share once, inherit forever.

  3. Stand up a project

    One command builds the whole thing — the Notion workspace, the project in the graph, and a wired CLAUDE.md in your repo. Preview first with --dry-run.

    npm run new:project -- --name "Acme" --dry-run   # preview, creates nothing
npm run new:project -- --name "Acme"             # for real

  4. Drive it with the toolkit

    Every skill has a command. A few to start:

    npm run resume                       # "where were we?" brief
npm run search -- "how do I make money"   # find by keyword (+ meaning, once embedded)
npm run lint:graph                   # integrity check
npm run export                       # versioned backup, committed to git

    The full list lives in the README toolkit table. Turning on search-by-meaning is one optional extra step — see Quickstart.

That's it — you're set up.

Whichever path you took, you now have a memory that grows with your work and a workspace to run it. Capture where it's natural; ask where it's powerful; let the assistant keep it honest.
