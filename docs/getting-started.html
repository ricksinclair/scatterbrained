getting started

One command. Then explore.

No graph-database knowledge needed. One command downloads Scatterbrained, starts the database, loads a demo graph, and opens the Studio.

$ npx scatterbrained@alpha studio # → http://localhost:4317

you'll need: Node 18+ · Docker (one thing, honestly: it hosts the bundled database) · ~2 minutes · no account, nothing in the cloud

what that command does

Run it — the demo graph loads by default.

On first run it finds a database or quietly starts one in Docker, applies the schema, and seeds a small fictional engineering team's graph so there's something to see — it never touches an existing graph's data. Then the Studio opens in your browser.

  1. Don't have Node or Docker?

    macOS, in Terminal (skip lines you already have):

    brew install node
brew install --cask docker   # then open Docker once so it's running

    no-GUI brew install colima docker then colima start works too. Windows & Linux notes are below.

  2. Run the one command

    npx scatterbrained@alpha studio          # → http://localhost:4317

    tip install it once with npm i -g scatterbrained@alpha, then it's just scatterbrained studio. Don't want the demo seed? SB_NO_DEMO=1.

  3. Take the in-app tour

    Click Help (the ? at the bottom of the left rail) → Start the tour — a narrated auto-play through the headline surfaces. It's also a command in the ⌘K palette. Prefer to read first? The same beats, as a page.

your first five minutes

Now that it's open.

  1. Read the brief, then the map

    The dock leads with what's new, due, and blocked. Drag to pan, scroll to zoom, click any node — the inspector on the right builds itself from what that node is.

  2. Ask the graph something

    Press ⌘K (or /) and type "what's due" — you get dated answers, not search hits. The left rail's lenses are one key each: G graph, T time, C code, A agents.

  3. Capture something

    Hit Capture (+) on the rail and paste any URL — it lands as a connected source and the camera flies to it. From a terminal: scatterbrained capture "a thought".

  4. Walk a decision's history

    In the demo graph, focus “Run self-managed Kafka for ingestion” — it's superseded, with the reason kept and the successor one click away. That's the rewind the whole thing is built around.

when you're ready — your own graph

Bring your own graph.

The demo is a loaner. Point the same command at a graph you already run by setting the connection env vars — Scatterbrained applies its schema idempotently and never deletes data. A fresh graph starts empty; you fill it as you work.

NEO4J_PASSWORD=… npx scatterbrained@alpha studio     # your own database (NEO4J_URI if it isn't local)

Prefer a clone? The repo is the same app plus the full graph toolkit:

git clone https://github.com/ricksinclair/scatterbrained.git
cd scatterbrained
npm install
npm start          # same dance: database, schema, demo, Studio
Windows

winget install OpenJS.NodeJS Docker.DockerDesktop, launch Docker Desktop, then the npx command (or npm install && npm start in a clone).

Linux

Node 18+ (distro or nvm) and Docker Engine, then the npx command. Already running a graph database? Set NEO4J_PASSWORD and skip Docker entirely.

upgrading

Your graph never lives in the app — it stays in the database, with config and models in their own homes on disk, so upgrading swaps only code and never touches your data. npx users get the newest alpha automatically on every run; a global install re-runs npm i -g scatterbrained@alpha; a clone pulls and reinstalls. The demo seed never re-runs on a graph that has data, and each version's story is in the changelog.

two ways in

Type a command, or just ask.

The published CLI is deliberately small — the four commands a new user needs. Everything else ships with the repo clone, and if you use Claude with the graph connected, plain language works too.

The CLI — exactly four commands

scatterbrained studio — launch (auto-starts the database + demo)

scatterbrained capture "a thought or a URL" — into the graph, with a receipt

scatterbrained status — is it up, what's in the graph, newest release

scatterbrained backup — your graph as JSON plus your memory as plain markdown, so leaving the app (or any AI tool) never costs you your knowledge

The full toolkit (lint:graph, resume, search, export, new:project, …) ships with the repo — clone it and run npm run <command>.

Ask Claude

If you use Claude with the graph connection, say what you want in plain language — no terminal.

"Where were we on the shipment project?"

"Capture this link and wire it to the pipeline decision."

honest, up front

Scatterbrained is alpha — surfaces and scripts can change between versions. Talking to it, and AI summaries / Q&A, run on a local model: the bundled Slipway runtime starts with the Studio and serves one once you pull it (tour, beats 12–14). Models come from one of two places, and both are a one-time download: Hugging Face — Slipway's Browse searches the public Hub catalog (no account, no key) and caches the model on your disk — or Ollama, if you run it, with models you've pulled via ollama pull. After the download the model loads and runs fully offline. Everything else works without a model, and nothing leaves your machine.

the words, in 60 seconds

A tiny glossary.

Knowledge graph
Your notes, decisions, files, and the people and projects they touch — stored as things and the links between them, so you follow a thread instead of hunting for a file.
The Studio
The app you actually use: a browser window that draws the graph as a living constellation, with a left rail of lenses — Graph · Time · Code · Agents — and a ⌘K palette that answers questions.
The database
A local graph database holds it all — started for you in Docker; you never have to touch it, and it exports to plain JSON in git. Your data never leaves your machine.
Protected facts & notes
Pin anything you want locked in — a number is the classic example, but a name, a date, or an exact phrase works too — and a rewrite can't quietly change it: alterations are held for your approval. Jot a note on anything and the next agent sync acts on it.
Record vs. intention time
When a fact was written or superseded is kept automatically; deadlines, revisit dates, and targets are yours to set — the Time lens reads them back as an agenda.
Slipway
Scatterbrained's bundled local-AI runtime — a small control panel that starts quietly alongside the Studio (port 8765) and serves the model when you talk to your graph. It loads nothing until you ask, browses and downloads models for you, and runs entirely on your machine. You'll meet it once, on first model download; after that it just works. SLIPWAY_AUTOSTART=0 turns it off entirely.
Hugging Face
The open library where the ML community publishes models — think "GitHub for AI models." Slipway's Browse searches its public catalog and downloads a model once into a cache on your disk (no account, no key, size shown before you pull); after that the model runs fully offline.
Ollama
A popular free app that runs AI models locally on any platform. Scatterbrained doesn't require it — but if you already use it, every model you've pulled with ollama pull shows up in Slipway's picker, ready to talk to your graph.
