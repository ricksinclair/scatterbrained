scatterbrained

/ˈskadərˌbreɪnd/  ·  adjective  — a mind full of scattered thoughts; wired together, they remember everything.

Scatterbrained turns everything you know — notes, decisions, files, projects — into a living star map on your own computer: open it each morning to see what needs you, capture any thought or link in one keystroke and watch it land as a glowing dot, and rewind to see exactly what you believed six months ago and why, with every claim tracing back to its source. No cloud, no subscription — it's yours.

$ npx scatterbrained@alpha studio

Node 18+ · Docker, for the bundled database · opens with a demo graph · the two-minute walkthrough

why it exists

Claude is the brain. Scatterbrained is the memory it deserves.

AI tools win every do-the-task comparison — that's what they're for, and Scatterbrained doesn't compete with them. It answers the other question, the one that gets worse the more you use them: what happens to what was learned? Five moments any heavy AI-tool user will recognize.

01

You asked the same question in March and June and got different answers. Which was right?

In Scatterbrained an answer doesn't get overwritten — it gets superseded, with a date and a reason, and the old one stays walkable. Scrub a decision's history and see what you believed in March, what replaced it in May, and why. Both clocks are kept: when things were recorded, and when you learned better.

02

Your assistant "remembered" something wrong — and you can't tell where it came from.

Here, every claim points back at the source that informs it — the page, the file, the talk it came from — so "where did that come from?" is a click, not an archaeology dig. And the numbers that matter can be pinned: when any rewrite — an AI polish, a sync, a careless edit — would alter a protected fact, the change is held for your approval instead of slipping through.

03

That great code review from six weeks ago — where is it now?

Not buried in a closed PR tab. A review here is a graph node pinned to a repo at a commit, and every line comment is a note that stays connected — to the decision it questioned, the project it belongs to, the person or agent who wrote it. Six weeks later it's still one hop from everything it touched, and whatever's unresolved waits in the review queue with everything else that needs a human.

04

What exactly did the agent know when it did that?

With a graph underneath, that's a question with an answer. Ask what's due, what changed, what's blocked — the same questions an agent's brief is built from — and you get dated items back, not vibes. When a session captures something, it lands as a source node with a visible receipt, so "what it knew" and "what it learned" are things you can open later, not a context window that evaporated.

honest: the Agents lens tracks that Brief → Session → Capture → Insight loop around a local agent terminal (the optional Slipway companion) when one is running — it's feature-detected and optional. Capture and the receipts work without it.

05

New machine, a month later — where was I?

The Studio opens on a daily brief: what's new, what's due, what's blocked since your last visit — with pick up where you left off pointing at the last thing you touched. The Time lens turns the graph's dates into an agenda: overdue, today, upcoming, worth revisiting. And because all of it is files and a local database on your own disk, "new machine" is a restore, not an act of remembering.

what's underneath

Boring on purpose, where it counts.

A real vocabulary Thirteen node types and a closed set of relationship types, with a linter that rejects drift — so traversals stay predictable years in.
Bi-temporal by default Record time (created / superseded) is kept automatically; intention time (due, revisit, target) is yours to set. Nothing is deleted — it's invalidated, with a reason.
Local-first, exportable The graph lives in a local Neo4j — a Docker container the one command starts for you — and exports to plain JSON in git. No cloud, no telemetry, no account.
LLM-optional Summaries and Q&A light up only if a local model is running. Every screen — and everything on this page — works without one.

documents — PDF, Word, Excel, Markdown — open in place on their nodes, so a cited claim traces to the page it came from without a detour.

do you need this?

Honestly — maybe not yet.

If you run one project and chat with an AI, you don't. Native memory and a well-kept project doc are genuinely fine at that scale. Scatterbrained earns its keep when that stops scaling — several projects, months of decisions, more than one agent touching the same knowledge — when "which file was that in?" becomes the question you ask most. Come back then; the graph will still be yours to start.

before you adopt — it's an alpha

Plainly stated.

Alpha software Surfaces, scripts, and the CLI can change between versions. Pin what you depend on.
It expects local infrastructure Node 18+, and Docker for the bundled database — or point it at a graph database you already run.
The published CLI is deliberately small Exactly three commands: studio, capture, status. The full toolkit ships with the repo — clone it and use npm run <command>.
Search is literal, for now The palette's fallback search is substring matching, not semantic. A miss feels like a dumb search box — that's honest, and it's on the roadmap.
