01 — the problem

Three tools, three blind spots.

Notion

A wonderful place to write things down — and a hopeless place to ask "what across all my work connects to this?" It can't traverse relationships.

A graph database

Traverses relationships beautifully. But nobody opens a terminal to journal their day in Cypher. The capture friction kills it.

Vector "memory"

An opaque blob of embeddings. You can't see why it recalls something, and you can't reach in and correct it when it's wrong.

02 — the idea

Two capture lanes. One canonical graph.

You keep capturing where it's comfortable — Notion pages, plain Markdown, PDFs, decks. An agent reads only what changed, extracts the entities, and writes them into a single Neo4j graph where every fact carries a link back to its source. Notion is the notebook; the graph is the mind.

03 — inside the graph

What the graph actually looks like.

Ten node types, joined by a small fixed vocabulary of edges. Every insight points at what it's about; every fact points back at the source that informs it. A slice:

Insight a synthesized takeaway
Source a page or file, with provenance
Project a thing you're building
Idea a possibility or open question
Rule a constraint (law, policy)
Goal an outcome with a timeframe
Skill a capability
Resource something to learn from
Person · Organization people & orgs

the edge vocabulary is closed: ABOUT · INFORMS · DERIVED_FROM · CONTAINS · REQUIRES · CONSTRAINS · TEACHES · DEPENDS_ON · ACHIEVED_BY · WORKS_AT · SUPPORTS — no inventing new ones, so traversals stay predictable.

04 — the human surface

Notion does double duty.

The graph is the agent's memory. Notion is the human window. The same workspace plays two roles: a capture lane (pages flow into the graph, §2) and a set of human-readable digests — plain-language surfaces for the non-technical stakeholders who'll never open your repo, graph, or CI.

That whole stack ships with Engram. One command — engram new-project — stands up the Notion workspace (four databases + status pages), registers the project in the graph, and drops a wired CLAUDE.md into the repo. These databases are digests written at milestones, not logs you hand-feed: the Changelog is the flagship (a plain-English release note per version), Test Run Metrics is a health snapshot, and each database describes what it's for so an empty one is never mistaken for neglected work. (A flaky-test tracker, Problem Tests, is an opt-in fifth for teams.) Your code, git, and the graph stay the day-to-day source of truth; Notion is the window onto it.

And setup is one authorization, ever: share a single root page with your Notion integration, and every project you stand up nests under it — workspace, databases, and a capture page for notes — all inheriting access. One root, every project; no page-by-page sharing.

Two ways in: just ask Claude (over the Notion MCP — no token, no terminal), or the CLI with a Notion token for headless / CI use. Same result; pick your lane.

05 — what you can just do

Skills, not commands.

Everything Engram does is a skill you reach for in plain language. If you use Claude with the Notion connection, just ask — no token, no terminal. Prefer the keyboard? Every skill has a one-line command. Same result; pick your lane.

Stand up a project

One move builds the whole thing: a Notion workspace (a plain-English changelog, docs index, a health snapshot, an optional task board, a capture page), the project registered in the graph, and a wired CLAUDE.md dropped in your repo.

Ask: “Set up a new Engram project called Acme.”

engram new-project --name "Acme"

Refresh the whole graph

Re-reads only what changed across Notion and your files, folds in new facts, surfaces fresh cross-project insights, checks integrity, and backs everything up — in one pass.

Ask: “Run a graph-sync.”

engram resume · lint:graph · export

Find anything by meaning

Ask in your own words. “How do I make money from this?” surfaces notes about revenue and pricing even when they never say it — keyword and meaning, blended and ranked.

Ask: “Search Engram for how this makes money.”

engram search "how do I make money"

Pick up where you left off

A “where were we” brief across every project — freshness, goals, the newest conclusions, and what's still open — so a new session starts already caught up.

Ask: “Where were we?”

engram resume

Keep it honest

When a fact is replaced, retire it — Engram marks it superseded instead of deleting, so the history survives. A review pass surfaces the stale candidates for you to confirm.

Ask: “Retire the old pricing decision; it changed.”

engram review:supersession · supersede

Surface undocumented decisions

Decisions the agent makes can live only in the graph, where a person never sees them. A review pass flags any decision with no human-readable page refreshed since — so the why reaches Notion and your docs, not just a query.

Ask: “Which decisions aren't written up yet?”

engram review:decisions

Assemble context to paste

Pull a clean, scoped context block — by project, domain, or tag — ready to drop into any chat, so the assistant reasons over your real knowledge, not a vague summary.

Ask: “Give me the context block for Acme.”

engram context --project Acme

The “ask Claude” path runs over the Notion + Neo4j MCPs as recipes the agent follows directly — the no-terminal twin of the toolkit. Full command reference in the README.

06 — what makes it different

Eight ideas doing the work.

  1. i

    Two lanes, one graph

    A Notion lane and a filesystem lane (Markdown, text, PDF, Word, PowerPoint) feed the same graph. Capture anywhere; the facts converge on their natural keys.

  2. ii

    Deterministic change-gating

    Hash and timestamp probes find only what actually moved — no LLM, no re-reading unchanged content. Cheap, reproducible, and it scales to thousands of files.

  3. iii

    Provenance on every fact

    Everything ingested gets a Source → INFORMS edge. You can always trace why the graph believes something — the thing vector memory can't show you.

  4. iv

    Invalidate, don't delete

    Superseded facts get a valid_until and a superseded_by instead of vanishing. History is kept; contradictions resolve by recency. A palimpsest, not an overwrite.

  5. v

    Lint-enforced integrity

    Rules become checks. lint:graph fails on orphans, undated nodes, unlinked insights, and off-vocabulary tags — so the graph can't silently rot.

  6. vi

    Agent-native & owned

    Built so an LLM resumes exactly where it left off and tends the graph itself. Plain Neo4j + scripts + git — no vendor, no framework to outgrow.

  7. vii

    Enforced recall — honestly costed

    New projects ship .claude/ hooks that re-inject the “consult the graph first” rule every turn, so the agent can’t quietly skip it — rules become checks, applied to memory, not just integrity. The honest cost: ~40 tokens of context per turn (the full rule is paid once per session). Review, tune, or switch it off via /hooks.

  8. viii

    Decisions reach humans, not just the graph

    A captured decision is a graph fact a person can’t casually read — so the human-readable docs can rot behind a rich graph (the inverse of the problem this set out to solve). review:decisions flags any decision with no Notion page, README, or site refreshed since — so it gets written up. The same enforcement as recall, pointed the other way: capture isn’t done until it’s in both the graph and a place a human reads.

What the enforcement costs your context — measured

Honesty about the one ongoing cost: only the graph-first recall hooks (vii) add standing context. The decision check (viii, review:decisions) is a command you run, not an injected prompt — it costs nothing per turn. The hooks are ~139 tokens once per session + ~40 per turn.
