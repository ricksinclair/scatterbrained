Documentation

What this is

A control panel for running an agent CLI against a local LLM on this Mac — start/stop models, swap models, and now swap which agent (Claude Code, OpenCode, Hermes Agent) runs them, all from one place.

The two commands

claude-localLaunches whichever launcher is selected (the Launcher dialog), against whichever model is selected. Starts the model server for you if needed. The name stuck around from before launchers existed — it now dispatches to Claude Code, OpenCode, or Hermes Agent.
mlx-controlOpens this control panel in your browser.

Launchers

Claude CodeAnthropic's CLI. Works with local + cloud models. The default.
OpenCodeMIT, multi-provider agent CLI. Local models only here — writes ~/.config/opencode/opencode.json pointing at whatever local server is active.
Hermes AgentNousResearch's agent CLI. Local models only here — writes ~/.hermes/config.yaml + .env. Install from the Launcher dialog (runs the official installer, ~3–5 min) or manually: curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash.

Cloud models currently only work through Claude Code — OpenCode/Hermes + Cloud isn't wired up yet (each has its own native cloud-provider system that's out of scope for now).

How the pieces fit

Claude Code ──▶ MLX    (local) : via vllm-mlx — Anthropic-native, no router
Claude Code ──▶ Ollama (local) : direct — Anthropic-native, no router
Claude Code ──▶ Cloud  (OpenRouter/DeepSeek/Nous) : via claude-code-router
Claude Code ──▶ Cloud  (Anthropic) : direct, with your API key

Two backends

mlxvllm-mlx serves a model from your HuggingFace cache on port 8080, speaking the Anthropic API natively (no router). Used for your exact Qwen3.6-35B-A3B. Loads in ~15–60s, ~19 GB RAM.
ollamaThe Ollama daemon on port 11434 loads/unloads models on demand. Many models available; snappier to switch.

Using the panel

  • Start / Stop / Restart — control the active model's server. Memory & PID show live while running.
  • Change model — pick any local model. If a server is running it restarts to load the new one; if stopped it just selects. The choice is saved so claude-local follows it. Each non-active model has a 🗑 to delete it and free disk (the active model is protected — switch away first).
  • Cloud — add API keys for OpenRouter, Anthropic, DeepSeek, or Nous/Hermes. Keys are stored locally in cloud-providers.json (chmod 600), shown masked, never logged. Enabled providers' models join the picker under Cloud; selecting one routes remotely (no local server, no RAM). OpenRouter alone covers most models (Hermes, Claude, DeepSeek, …) with one key.
  • Quit panel — closes this web app. Choose Quit panel only (model keeps running) or Stop server & quit (frees RAM).

Adding more models

  • Browse HF (in the model card) — search HuggingFace for MLX-ready models and download them in-app. It shows each model's size and checks it against free disk before downloading (keeps a 10 GB buffer), with a live progress bar. Done downloads appear in “Change model” automatically.
  • Ollama — ollama pull <name> in a terminal; it shows up in the picker too.

Troubleshooting

  • Stuck on "Starting…" — big MLX models take 30–60s. Check the Activity log for errors.
  • Free memory now — Stop in the panel, or pkill -f "vllm-mlx serve" / ollama stop <model>.
  • Go back to real Claude — just run claude. This setup never touches it.
  • Panel won't reopen — pkill -f mlx-control/server.py then mlx-control.

Caveats

Local models — especially 4-bit quantized ones — are slower and noticeably weaker at agentic tool-use than hosted Claude. Great for chat and simple edits; rough for complex multi-step work. The llama3.1:8b and qwen2.5-coder:1.5b models are the quick ones.

Built on open components: Ollama (MIT), vllm-mlx (Apache-2.0), claude-code-router (MIT, kept for cloud providers). Full write-up in ~/Projects/mlx-control/README.md.

Terminals

No terminal sessions. Pick an agent CLI and click + Launch to run it against the selected model, or + Shell for a plain shell.
